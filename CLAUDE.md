## Project Overview

Unity Tower Defense game built with Unity 6000.0.69f1 (LTS) using C# and Universal Render Pipeline (URP 17.0.4).

## Development Environment

- **Unity Version:** 6000.0.69f1 ‚Äî must use this exact version to open the project
- **IDE:** Visual Studio with "Microsoft.VisualStudio.Workload.ManagedGame" workload
- **Solution file:** `My project.sln` (auto-generated by Unity)

## Build & Run

All build/run operations go through the Unity Editor ‚Äî there are no CLI build commands configured:

- **Play in Editor:** Open project in Unity, press the Play button
- **Build executable:** File > Build Settings > Build (targets Windows Standalone)
- **Testing:** Unity Test Runner (Window > General > Test Runner); `com.unity.test-framework` is installed but no tests currently exist

## Architecture

### Manager Singletons

All managers use the Unity singleton pattern (`public static ManagerClass instance`):

| Manager | Responsibility |
|---|---|
| `GameManager.cs` | Gold, lives, wave count, game over state |
| `BuildManager.cs` | Tower selection and placement orchestration |
| `WaveManager.cs` | Spawns enemies in waves (starts at 3, +2 per wave) |
| `UIManager.cs` | Updates TextMeshPro UI elements |
| `PauseMenu.cs` | Pause/resume, volume, scene navigation |

### Game Entities

- **`Tower.cs`** ‚Äî Finds nearest enemy within range, shoots `Bullet` prefabs at `fireRate`; configured via `TowerData` ScriptableObject
- **`TowerData.cs`** ‚Äî ScriptableObject holding tower stats (name, cost, range, fireRate, damage, color); three assets exist: `TorreBasica`, `TorreCa√±on`, `TorreFrancotirador`
- **`Enemy.cs`** ‚Äî Follows `WaypointPath` waypoints, reduces player lives on completion, awards gold on death
- **`Bullet.cs`** ‚Äî Tracks a target enemy, applies damage on collision
- **`BuildSlot.cs`** ‚Äî Clickable terrain tile; on click, calls `BuildManager` to place the selected tower
- **`SleepCharacter.cs`** ‚Äî Purchasable character (150 gold) that generates 5 gold every 10 seconds when awake

### World Systems

- **`WaypointPath.cs`** ‚Äî Singleton; child `Transform`s define the ordered enemy path; exposes `Waypoints` array
- **`CameraController.cs`** ‚Äî WASD/arrows to pan, Q/E to rotate, R/F for height, scroll wheel to zoom

### Key Dependencies (Packages/manifest.json)

- `com.unity.ai.navigation` ‚Äî NavMesh for enemy pathfinding
- `com.unity.inputsystem` ‚Äî New Input System for camera and UI input
- `com.unity.render-pipelines.universal` ‚Äî URP graphics
- `com.unity.timeline` ‚Äî Animation timeline support

### Scenes

- `Scenes/MainMenu.unity` ‚Äî Main menu (`MainMenu.cs` handles scene loading)
- `Scenes/SampleScene.unity` ‚Äî Gameplay scene containing all game objects

### URP Quality Presets

Two render pipeline asset pairs configured in `ProjectSettings/`:
- `Mobile_RPAsset` / `Mobile_Renderer` ‚Äî optimized for mobile
- `PC_RPAsset` / `PC_Renderer` ‚Äî high-quality for PC
